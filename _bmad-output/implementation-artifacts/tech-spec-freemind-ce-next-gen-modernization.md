# Tech-Spec: FreeMind CE Next-Gen Modernization — Infrastructure to Feature Excellence

**Created:** 2026-03-11

## Overview

### Problem Statement

FreeMind CE has completed its core migration (JAXB, UTF-8, plugins, CI/CD, HTML export, Context Graph) but the foundation has critical gaps that block sustainable development:

1. **Tests never run:** 24 test files sit in the `main` sourceSet — Gradle compiles them as production code but never executes them as tests. `compileTestJava` reports `NO-SOURCE`.
2. **CI has no quality gate:** `build.yml` runs `./gradlew build` but never `./gradlew test`. No static analysis, no coverage, no vulnerability scanning.
3. **No release automation:** Releases require manual tagging. No automated changelog, no Conventional Commits enforcement, no versioning automation.
4. **Limited distribution:** Only DMG/EXE/DEB via jpackage. No package manager presence (Homebrew, Winget, Flatpak). No code signing — macOS shows "unidentified developer", Windows shows SmartScreen warning.
5. **Technical debt:** HotEqn (GPLv3 license conflict), Lucene 4.6 (2013), 75 bitmap PNG icons (no HiDPI/dark mode), missing i18n keys.
6. **Repo lacks polish:** README is functional but uninspiring for a project with 20 years of history. No CONTRIBUTING.md, no issue templates, no social preview.

The goal: preserve the beloved 20-year-old user experience while modernizing the entire developer experience, sustainability infrastructure, and AI-assisted development workflow to the highest standards.

### Solution

Multi-phase modernization with infrastructure-first approach:

1. **v1.1.x** — Fix test infrastructure, harden CI/CD, automate releases, polish repo aesthetics
2. **v1.2.0** — Expand to additional package formats (RPM, MSI, AppImage, portable ZIP)
3. **v1.3.0** — Modernize LaTeX (HotEqn→JLaTeXMath), activate exports, add JSON/YAML export
4. **v1.4.0** — SVG icons, FlatLaf theme refinement, visual polish
5. **v1.5.0** — Fill i18n gaps, add accessibility, optimize performance

### Scope

**In Scope:**

1. **v1.1.x — Infrastructure, DX & Automation (Foundation)**
   - Test sourceSet fix (move `tests/` from `main` to proper `test` sourceSet)
   - JUnit 5 migration (24 existing tests)
   - Test dependencies as `testImplementation` (currently `implementation`)
   - CI test enforcement (build.yml test step, test reporter)
   - Pre-commit test hook
   - JaCoCo coverage reporting
   - SpotBugs or Error Prone static analysis
   - OWASP dependency vulnerability scanning
   - Conventional Commits + commitlint
   - release-please integration (automated versioning, Release PR, CHANGELOG.md)
   - Consistent release asset naming
   - README.md enrichment (banner, screenshots, badges, feature list)
   - CONTRIBUTING.md, issue/PR templates, social preview
   - jpackage enhancements (file-associations for .mm, license-file)

2. **v1.2.0 — Multi-Platform Distribution**
   - Additional formats: MSI (Windows), RPM (Linux), AppImage, portable ZIP/tar.gz

3. **v1.3.0 — LaTeX & Export Modernization**
   - HotEqn → JLaTeXMath (BSD license, AMS math support)
   - LaTeX export activation (article, book, beamer — currently commented out)
   - JSON/YAML export
   - Dead export cleanup (Applet, Flash)

4. **v1.4.0 — Visual Modernization**
   - SVG icon set (75 PNG → vector, HiDPI, dark mode variants)
   - FlatLaf theme refinement
   - Splash screen modernization

5. **v1.5.0 — Quality & Dependency Updates**
   - i18n gaps (FlatLaf, Context Graph, new features)
   - Accessibility (Java A11y API)
   - Performance optimization (lazy loading for large maps)

**Deferred (future milestones):**

- Package managers (Homebrew Cask, Winget, Flatpak, Scoop, AUR) — after community traction
- Code signing (macOS notarization, Windows Authenticode, GPG) — requires paid developer accounts
- Markdown import plugin (.md → .mm converter) — nice-to-have, not urgent
- Markdown export improvement (richer .md output — tables, links, images) — current export works
- Lucene 4.6 → 9.x migration — large API change, current search works fine

**Out of Scope:**

- Inline formula support (`$...$` syntax in nodes) — cancelled, requires deep NodeView rendering changes disproportionate to current architecture
- v2.0+ vision items (WebSocket collaboration, cloud storage, web viewer, mobile app, plugin marketplace)
- Jabber/XMPP plugin (deprecated, security risk)
- Database plugin Gradle integration (low demand)
- Full UI/UX redesign beyond icon and theme updates
- Groovy SecureASTCustomizer (low practical risk — users run their own scripts)

### Technical Decisions (Updated After Investigation)

- **FreeMindMainMock refactoring required:** Extract to production package before test sourceSet separation — prerequisite #1
- **JUnit 3 → JUnit 5 directly (skip JUnit 4):** Tests are JUnit 3 (TestCase), use JUnit 5 vintage engine for gradual migration
- **release-please over semantic-release:** No Node.js dependency, designed for GitHub Actions
- **SpotBugs over PMD/Error Prone:** Lower false-positive rate for legacy codebases
- **JLaTeXMath 1.0.7:** BSD license, matches Freeplane's proven version (`org.scilab.forge:jlatexmath`)
- **Conventional Commits:** Required for release-please automated changelog
- **Infrastructure first:** Test fix → CI hardening → release automation → then features
- **Property tests: rewrite not re-enable:** Existing ones reference non-existent APIs
